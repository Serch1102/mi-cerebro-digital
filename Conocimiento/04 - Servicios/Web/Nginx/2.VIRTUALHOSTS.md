### VirtualHosts Basados en Dominios

==Los VirtualHosts basados en dominios permiten que un servidor web responda a múltiples nombres de dominio en una misma dirección IP==. ==Cada VirtualHost está configurado para servir un sitio web específico asociado a un nombre de dominio.==

En la configuración de Apache, por ejemplo, un VirtualHost basado en dominios se vería así:

```apache
<VirtualHost *:80>
    ServerName www.ejemplo1.com
    DocumentRoot /var/www/ejemplo1
    # Otras configuraciones específicas del sitio 1
</VirtualHost>

<VirtualHost *:80>
    ServerName www.ejemplo2.com
    DocumentRoot /var/www/ejemplo2
    # Otras configuraciones específicas del sitio 2
</VirtualHost>
```

Cada bloque `<VirtualHost>` contiene la configuración específica para un sitio web, incluido el nombre de dominio y la ruta del documento raíz.

### VirtualHosts Basados en IPs

==Los VirtualHosts basados en IPs permiten que un servidor web responda a múltiples direcciones IP en una misma máquina física==. Cada dirección IP puede tener su propio conjunto de configuraciones de servidor y alojar diferentes sitios web.

En la configuración de Apache, un VirtualHost basado en IPs se vería similar al ejemplo anterior, pero especificando la dirección IP en lugar del nombre de dominio.

```apache
<VirtualHost 192.168.1.1:80>
    ServerName www.ejemplo1.com
    DocumentRoot /var/www/ejemplo1
    # Otras configuraciones específicas del sitio 1
</VirtualHost>

<VirtualHost 192.168.1.2:80>
    ServerName www.ejemplo2.com
    DocumentRoot /var/www/ejemplo2
    # Otras configuraciones específicas del sitio 2
</VirtualHost>
```

En ambos casos, ==cada bloque `<VirtualHost>` define la configuración para un sitio web específico, permitiendo que un servidor web aloje múltiples sitios y responda adecuadamente a las solicitudes entrantes según el nombre de dominio o la dirección IP solicitada==.

### Paso 1: Crear el Directorio del Sitio Web

Primero, se crea un directorio para almacenar los archivos del sitio web:

```bash
sudo mkdir -p /var/www/user.com/html
sudo chmod -R 755 /var/www/user.com
```

### Paso 2: Asignar Propiedad del Directorio

Asegúrate de que el directorio pertenezca al usuario correcto:

```bash
sudo chown -R $USER:$USER /var/www/user.com/html
```

El comando `chown` se utiliza para cambiar el propietario y el grupo de un archivo o directorio en el sistema. La opción `-R` se utiliza para realizar la operación de manera recursiva, lo que significa que se aplicará a todos los archivos y subdirectorios dentro del directorio especificado.

- `sudo`: Se utiliza para ejecutar el comando con privilegios de superusuario, lo que permite realizar cambios en archivos y directorios que requieren permisos especiales.
    
- `$USER:$USER`: `$USER` es una variable de entorno que representa al usuario actual que está ejecutando el comando. En este caso, el usuario que está configurando el VirtualHost. ==La notación `usuario:grupo` especifica el nuevo propietario y grupo del directorio. Al utilizar `$USER` dos veces, se establece tanto el propietario como el grupo del directorio como el usuario actual.==
    
- `/var/www/user.com/html`: Es la ruta del directorio que se está creando para almacenar los archivos del sitio web. Esta ruta puede variar dependiendo de la configuración del servidor.
    

==El propósito de este paso es asegurarse de que el usuario que está configurando el VirtualHost tenga los permisos adecuados para acceder y modificar los archivos dentro del directorio del sitio web==. Al asignar la propiedad del directorio al usuario actual, se garantiza que el usuario tenga control total sobre los archivos y pueda realizar las operaciones necesarias, como escribir y modificar archivos de configuración, subir archivos al servidor, etc.

==Es importante asignar los permisos adecuados a los archivos y directorios del servidor web para garantizar la seguridad y el funcionamiento adecuado del sitio web.==


### Paso 3: Crear un Archivo de Configuración de Nginx

Se crea un nuevo archivo de configuración para el VirtualHost:

```bash
sudo nano /etc/nginx/sites-available/user.com
```

Se incluye la configuración básica en el archivo `user.com`:

```nginx
server {
    listen 80;
    server_name user.com www.user.com;

    location / {
        root /var/www/user.com/html;
        index index.html index.htm;
    }

    error_page 404 /404.html;
    location = /404.html {
        root /var/www/user.com/html;
    }

    error_page 500 502 503 504 /50x.html;
    location = /50x.html {
        root /var/www/user.com/html;
    }
}
```

- `error_page`: Esta directiva ==define qué página se mostrará cuando ocurra un determinado error==. Por ejemplo, `error_page 404 /404.html;` indica que cuando se genere un error 404, Nginx debe mostrar la página `/404.html`.
    
- `location`: ==Define la ubicación del archivo de la página de error especificada==. Por ejemplo, `location = /404.html { root /var/www/user.com/html; }` indica que la página de error 404 se encuentra en el directorio `/var/www/user.com/html`.

-  1. **error_page 500 502 503 504 [[Glosario#^13889f|/50x.html]];** ^10839a
    
    - Similar al caso anterior, esta línea indica que cuando se produzcan errores 500, 502, 503 o 504 (errores del servidor), Nginx debe redirigir la solicitud al archivo HTML `[[Glosario^|50x.html]]/`.
    
1. **location = /50x.html { root /var/www/user.com/html; }**
    
    - Esta directiva de ubicación establece la ruta del archivo HTML que se utilizará para manejar los errores 500, 502, 503 y 504. Indica que cuando se solicite la página `/50x.html`, Nginx debe buscar este archivo en la ruta `/var/www/user.com/html`.

### Paso 4: Habilitar el Archivo de Configuración

Se habilita el VirtualHost creando un enlace simbólico:

```bash
sudo ln -s /etc/nginx/sites-available/user.com /etc/nginx/sites-enabled/
```
- `ln`: Este es el comando para crear enlaces, tanto duros como simbólicos.
    
- `-s`: Esta opción indica que se está creando un enlace simbólico. Los enlaces simbólicos son referencias a otros archivos o directorios, similar a los accesos directos en sistemas Windows. No contienen los datos del archivo original, solo un enlace a su ubicación.
### Paso 5: Probar la Configuración de Nginx

Antes de reiniciar Nginx, se prueba la configuración para verificar posibles errores de sintaxis:

```bash
sudo nginx -t
```
- `-t`: Esta opción le indica a Nginx que realice una prueba de sintaxis en sus archivos de configuración. Nginx analizará los archivos de configuración para verificar si hay errores de sintaxis o de configuración.
### Paso 6: Reiniciar Nginx

Finalmente, se reinicia Nginx para aplicar los cambios:

```bash
sudo systemctl restart nginx
```

### Paso 7: Editar el Archivo Hosts Local (Opcional)

Si no se ha configurado DNS, se puede editar el archivo hosts local para probar el VirtualHost:

```plaintext
# Añadir esta línea a /etc/hosts en sistemas Unix o
# C:\Windows\System32\Drivers\etc\hosts en Windows
127.0.0.1 tudominio.com
```
